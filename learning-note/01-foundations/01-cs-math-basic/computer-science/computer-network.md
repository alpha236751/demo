# 第三章 网络层（Network Layer）
## 3.1 网络层基础概念
### 3.1.1 概念定义【考研重点】【面试高频】
1. **网络层(Network Layer)**：TCP/IP五层、OSI七层模型第三层，介于数据链路层与传输层之间，核心职责实现**跨不同网段主机端到端寻址、分组转发、路由选路**，屏蔽底层物理链路、链路层协议差异。
2. **分组(Packet)**：网络层数据封装单元，又称IP分组，链路层封装后变为帧(Frame)。
3. **路由(Routing)**：依据路由表选择最优转发路径的过程，分为静态路由、动态路由两类。

### 3.1.2 底层原理
网络层基于**逻辑寻址+路由查表**工作：
1. 源主机凭借目标IP地址，区分目标主机是否位于本地网段；
2. 同网段：数据包直接交付，交由数据链路层ARP寻址转发；
3. 跨网段：数据包转发至网关路由器，路由器剥离二层帧头，解析IP首部提取目的IP，查询本机路由表匹配出下一跳地址，重新封装二层头部向下转发，逐跳传递直至目的主机。

### 3.1.3 文字框图（报文封装流转）
```
上层传输段(Segment)
    ↓ 增加IP首部 → IP分组(Packet)【网络层PDU】
IP分组
    ↓ 增加帧首部+尾部 → 数据帧(Frame)【链路层PDU】
数据帧
    ↓ 比特流编码 → 物理层Bit数据流
```
跨路由器报文处理框图：
```
入方向：比特流→剥帧头→提取IP报文→路由查表
出方向：选定下一跳→重封装二层帧→比特流发送
```

### 3.1.4 关键对比表格
|对比项|同网段数据交付|跨网段路由转发|
| ---- | ---- | ---- |
|寻址依据|MAC地址(硬件地址)|IP地址(逻辑地址)|
|转发设备|交换机(Switch，二层设备)|路由器(Router，三层设备)|
|IP地址变化|源、目的IP全程不变|源/目的IP始终不变，二层MAC逐跳替换|
|路由参与|无需查询路由表|逐跳查询各路由器路由表|

### 3.1.5 易错坑点
1. 误区：路由器修改IP地址；纠正：**IP全程不变，仅二层源目MAC不断更换**【面试高频易错】。
2. 误区：交换机可跨网段通信；纠正：交换机仅二层转发，无IP寻址、路由能力。
3. 误区：ARP协议工作在网络层；纠正：ARP隶属于数据链路辅助协议，逻辑依附二层。

## 3.2 IP协议(Internet Protocol)【考研重点】【面试高频】
### 3.2.1 概念定义
**IP(网际协议)**：网络层核心协议，分为IPv4、IPv6两个版本，提供**无连接、不可靠**的分组投递服务，负责IP编址、分片重组、路由寻址。

### 3.2.2 底层原理
1. 无连接：通信前无需建立握手连接，每个IP分组独立选路，路径可不一致；
2. 不可靠：IP无重传、流量控制、差错重传机制，出错直接丢弃报文，差错补救交由上层TCP实现；
3. IP分片：链路MTU小于IP分组长度时，路由器拆分IP报文为多个分片，目标主机接收后依据标识、片偏移重组。

### 3.2.3 文字框图（IPv4首部简易结构）
```
0~3bit：版本 | 4~7bit：首部长度 | 8~15bit：服务类型(TOS)
16~31bit：总长度
32~47bit：标识 | 3bit标志位 | 13bit片偏移
生存时间TTL(8bit) | 协议号(8bit) | 首部校验和(16bit)
源IP地址(32bit)
目的IP地址(32bit)
可选字段(可变) + 填充位
```

### 3.2.4 关键对比表格
|参数|IPv4|IPv6|
| ---- | ---- | ---- |
|地址长度|32bit(4字节)|128bit(16字节)|
|分片实现|路由器可分片|仅源主机分片，路由器禁止分片|
|广播地址|支持全网广播|无广播，使用组播替代|
|首部校验|IP首部自带校验和|取消IP首部校验，交由上下层校验|

### 3.2.5 易错坑点
1. TTL误区：TTL代表链路跳数，非时间，每经过一台路由器TTL-1，TTL=0直接丢包；
2. IP分片误区：只有目的主机负责重组分片，中间路由器不重组。

## 3.3 配套辅助协议：ARP/ICMP【面试高频】【常识了解】
### 3.3.1 概念定义
1. **ARP(地址解析协议 Address Resolution Protocol)**：实现IP→MAC地址映射；
2. **ICMP(网际控制报文协议 Internet Control Message Protocol)**：依附IP协议，用于差错报告、网络连通性探测（ping命令底层）。

### 3.3.2 底层原理
1. ARP原理：主机发送广播ARP请求帧，同网段所有主机接收，匹配目标IP主机单播回复自身MAC，本机缓存ARP映射表；
2. ICMP原理：IP报文异常丢弃时，源路由器/主机封装ICMP差错报文返回源IP；ping依靠ICMP Echo请求/应答报文探测连通。

### 3.3.3 文字框图
ARP交互流程：
```
主机A：广播ARP请求（我是IP_A，查询IP_B的MAC）
网段所有主机接收，主机B匹配IP_B
主机B：单播ARP应答（IP_B-MAC_B）→主机A更新ARP缓存
```
PING报文流转：
```
客户端发出ICMP Echo Request→网关路由转发→目标主机返回ICMP Echo Reply
```

### 3.3.4 关键对比表格
|协议|工作层级|报文承载|核心作用|
| ---- | ---- | ---- | ---- |
|ARP|链路层附属|封装以太网帧|IP解析MAC|
|ICMP|网络层附属|封装在IP分组内|差错、连通探测|

### 3.3.5 易错坑点
1. 误区：ARP跨网段解析MAC；纠正：ARP广播无法跨路由器，跨网段ARP只能解析网关MAC；
2. 误区：ICMP是独立三层协议；纠正：ICMP数据作为IP数据字段封装，无独立首部。

## 3.4 路由协议基础【考研重点】【常识了解】
### 3.4.1 概念定义
**路由协议(Routing Protocol)**：路由器之间交换路由条目、自动生成路由表的协议，分为内部网关协议IGP、外部网关协议EGP。

### 3.4.2 底层原理
同一自治域内IGP互相交换网段信息，通过算法计算开销生成最优路由；不同自治域依靠EGP交换跨域路由。

### 3.4.3 文字框图
```
自治系统AS1(IGP：RIP/OSPF) ←BGP(EGP)→ 自治系统AS2(IGP：OSPF)
AS内部：路由协议同步路由表
AS之间：BGP传递跨域路由
```

### 3.4.4 关键对比表格
|路由类型|静态路由|动态路由|
| ---- | ---- | ---- |
|配置方式|管理员手动配置|路由协议自动学习|
|拓扑变更|需手动修改路由|自动收敛更新路由|
|适用场景|小型简单网络|大型广域网、互联网骨干网|

### 3.4.5 易错坑点
1. 误区：交换机运行OSPF；纠正：OSPF为三层路由协议，仅路由器/三层交换机支持。
