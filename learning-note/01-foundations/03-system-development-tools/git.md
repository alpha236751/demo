### 一、Git简介

两种版本控制系统 
- **集中式版本控制系统** : SVN、CVS
- **分布式版本控制系统** : Git 

### 二、Git安装

官网安装 ：https://git-scm.com/downloads

### 三、Git使用方式

- 命令行(CLI)
- 图形化界面(GUI)
- IDE插件

### 四、Git初始配置

- 配置全局用户名和邮箱：`git config --global user.name "你的名字"`
- 配置全局邮箱：`git config --global user.email "你的邮箱"`
- 查看配置：`git config --list`

### 五、Git仓库（Repository）

- `git init <目录>` 初始化一个Git仓库，默认在当前目录
- `git clone <URL>` 克隆一个远程Git仓库到本地目录。


### 六、Git常用命令

#### 添加和提交文件

- `git status` 查看当前仓库文件的状态
  - 未追踪(untracked) 
  - 已修改(modified, changes not staged for commit) 
  - 已暂存(staged, changes to be committed)
  - 未修改(unmodified, nothing to commit, working tree clean)
  - `-s` 查看简略状态 (short format)

- `git add <文件>` 从工作区添加文件到暂存区，准备提交。
  - `.` 添加当前目录的所有文件到暂存区

- `git rm --cached <文件>` 删除暂存区的文件

- `git commit -m "提交信息"` 提交暂存区的修改到本地仓库。

- `git log` 查看本地仓库的提交历史。
  - `--oneline` 显示简洁的提交历史信息

#### 回退版本

- `git reset <版本>`  回滚到指定版本 默认最新head
  - `--soft` 不改变工作区和暂存区
  - `--mixed` 不改变工作区，改变暂存区(默认)
  - `--hard` 改变工作区和暂存区

- `git reflog` 查看本地操作历史记录 (reference log)

#### 查看差异

- `git diff` 什么都不加 默认查看工作区与暂存区的差异
  - `版本库` 加版本库 查看工作区与版本库的差异
  - `--cached 版本库` 查看暂存区与版本库的差异
  - `版本库1` `版本库2`  查看两个版本的差异
  - `文件名` 查看指定文件的差异
  - `分支1` `分支2` 查看两个分支的差异

#### 删除文件

- `git ls-files` 查看所有被追踪的文件
  - `--cached` 查看暂存区的文件(默认) -c
  - `--stage` 查看暂存区的文件(详细) -s
  - `--deleted` 查看已删除但未add -d
  - `--modified` 查看已修改但未add -m
  - `--unmerged` 查看未合并的文件 -u
  - `--others` 查看未追踪的文件 -o

- `rm <文件>` 从工作区删除文件
- `git rm <文件>` 从暂存区和工作区都删除文件并提交删除 
 - `--cached` 从暂存区删除文件

#### 忽略文件

- 应该忽略的文件
  - 自动生成的文件
  - 编译产生的中间文件
  - 日志缓存和临时文件
  - 敏感信息的配置文件

- `.gitignore` 忽略文件 放在项目根目录下


> 不会忽略已被追踪的文件

#### 远程仓库

- `git branch -m main` 修改当前分支名为main (move)

- `git remote add <远程仓库别名> <URL>` 添加远程仓库 默认origin
- `git remote -v` 查看远程仓库的别名和URL

- `git clone <远程地址HTTPS/SSH>` 从远程下载仓库到本地
  - `-b <远程分支>` 指定克隆分支
  - 最后可以指定克隆后的本地文件名，默认与远程同名 
  
- `git push` 推送本地分支到远程仓库
  - `-u`/`--set-upstream` 设置远程分支为默认分支，后续推送时无需指定远程分支
  - `<远程仓库名>` 推送指定远程仓库 默认origin
  - `<远程分支>:<本地分支>` 推送指定远程分支 默认找本地分支同名 本地分支默认为当前分支

- `git pull` 从远程仓库拉取分支 自动合并
  - `<远程仓库名>` 拉取指定远程仓库 默认origin
  - `<远程分支名>` 拉取指定远程分支合并到当前分支 默认找当前分支同名

- `git fetch` 从远程仓库拉取 只下载不合并
  - `--prune` 删除本地分支中不存在的远程分支
  - `<远程仓库名>`/`-a`/`--all` 拉取所有关联的远程仓库 默认只拉origin
  - `<远程分支名>` 拉取指定远程分支

#### 分支

- `git branch` 查看所有本地分支
  - `-a` 查看所有分支，包括远程分支和本地分支
  - `-r` 查看所有远程分支

- `git branch <分支名>` 创建新分支
- `git branch -d <分支名>` 删除指定分支,分支必须已合并到其他分支
- `git branch -D <分支名>` 强制删除指定分支

- `git checkout <分支名>/<版本id>` 切换到指定分支 可能与切换版本冲突
  - `-b <新分支名>` 创建新分支
- `git switch <分支名>` 切换到指定分支 
- `git merge <分支名>` 合并指定分支到当前分支
- `git rebase <分支名>` 变基当前分支到指定分支,没有合并冲突

### 七、SSH协议

- **SSH** 全称：Secure Shell（安全外壳协议），是一种加密的**远程登录 / 远程管理协议**，用来在网络上安全地操控另一台电脑（服务器、虚拟机、树莓派等）。

- SSH 密钥对基于**非对称加密算法**（主流为 RSA、Ed25519）构建，该体系包含两类数学关联但功能互异的密钥：
- **公钥**（Public Key）：对外公开，可自由分发，作为加密密钥 / 验证密钥；
- **私钥**（Private Key）：持有者专属，需严格保密，作为解密密钥 / 签名密钥。
- 二者由同一算法生成，构成密钥对，数学上满足：**公钥加密的数据仅能由对应私钥解密，私钥生成的数字签名仅能由对应公钥验证。**

- `ssh-keygen -t ed25519` 生成密钥对 默认RSA密钥类型
  - `-f <文件名>` 指定私钥保存路径和文件名，公钥会放在同目录的.pub文件 默认`~/.ssh/密钥类型`文件
